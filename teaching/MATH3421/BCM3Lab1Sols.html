Question 1 (Random walk Metropolis)

Consider a \(\text{N}(0,1)\) target, a random walk proposal with \(\text{U}(-a,a)\) innovations and the following function:

norm<-function(N,a) 
{
 vec <- rep(0,N)  # store samples here
 theta <- 0 # initial value
 vec[1] <- theta # store initial value
 for (i in 2:N) 
 {
  innov <- runif(1,-a,a) # generate innovation
  can <- theta + innov   # proposal
  laprob <- dnorm(can,log=TRUE)-dnorm(theta,log=TRUE) # log of acc. prob.
  u <- runif(1)
  if (log(u) < laprob){ 
   theta <- can # accept
  }
   vec[i] <- theta #store most recent theta
  }
 return(vec) #return vector of samples
}

Note that the proposal density is \(q(\theta^*|\theta)=1/2a\), for \(\theta-a<\theta^*<\theta+a\) and 0 otherwise.

Question 2 (A simple Metropolis-Hastings independence sampler)

Consider a \(\text{Gamma}(a,b)\) target, normal proposal (with mean and variance to match the target). The sampler is coded via the following function.

gammaMH<-function(N,a,b) 
{
 mu <- a/b; sig <- sqrt(a/(b*b))
 vec <- rep(0,N)
 theta <- a/b
 vec[1] <- theta
 for (i in 2:N) 
 {
  can <- rnorm(1,mu,sig)
  #split acc. prob. for easier display
  laprob1 <- dgamma(can,a,b,log=TRUE)-dgamma(theta,a,b,log=TRUE)
  laprob2 <- dnorm(theta,mu,sig,log=TRUE)-dnorm(can,mu,sig,log=TRUE)
  laprob <- laprob1+laprob2
  u <- runif(1)
  if (log(u) < laprob){ 
   theta <- can
  } 
   vec[i] <- theta
  }
 return(vec)
}
  1. Try \(a=0.1\), \(b=0.01\).

  1. Modify the sampler to also return the log of the target density and proposal density at the current value of the chain.
gammaMH<-function(N,a,b) 
{
 mu <- a/b; sig <- sqrt(a/(b*b))
 mat <- matrix(0,nrow=N,ncol=3) #add extra columns to store \log\pi and \log q
 theta <- a/b
 mat[1,] <- c(theta,dgamma(theta,a,b,log=TRUE),dnorm(theta,mu,sig,log=TRUE))
 for (i in 2:N) 
 {
  can <- rnorm(1,mu,sig)
  laprob1 <- dgamma(can,a,b,log=TRUE)-dgamma(theta,a,b,log=TRUE)
  laprob2 <- dnorm(theta,mu,sig,log=TRUE)-dnorm(can,mu,sig,log=TRUE)
  laprob <- laprob1+laprob2
  u <- runif(1)
  if (log(u) < laprob){ 
   theta <- can
  } 
   mat[i,] <- c(theta,dgamma(theta,a,b,log=TRUE),dnorm(theta,mu,sig,log=TRUE))
  }
 return(mat)
}

mat <- gammaMH(10000,0.1,0.01)
#Trace plots
par(mfrow=c(3,1))
plot(ts(mat[,1]),ylab="theta",xlab="Iteration")
plot(ts(mat[,2]),ylab="log pi theta",xlab="Iteration")
plot(ts(mat[,3]),ylab="log q theta",xlab="Iteration")

Comment 1: when the chain gets stuck at values of \(\theta\) around 0, the corresponding \(\pi(\theta)\) values are large relative to \(q(\theta)\) values. This leads to the contribution of \(q(\theta) / \pi(\theta)\) in the acceptance probability being very small, in turn leading to a very small acceptance probability \(\alpha(\theta^*|\theta)\), for typical values of \(\theta^*\). Hence, we get periods consisting of several (tens of) iterations for which the chain doesnt move. We refer to this phenomenon as `stickiness.

Comment 2: the chain gets stuck at large values of \(\theta\) due to the small values of the corresponding \(q(\theta)\). That is, these values are likely under the target but not the proposal. (This doesnt happen so often so you might need a longer run to see this.)

Question 3 (Crude oil volatility analysis)

Here, we repeat the analysis from problems class 1, but with a prior that precludes a conjugate analysis! Hence the need for Metropolis-Hastings.

We are given the following function:

#Function to evaluate \log\pi(\psi|y) up to an additive constant

lpost<-function(psi)
{
 lprior <- dnorm(psi,log=TRUE)
 llike <- -25*psi-0.5*0.0305/exp(2*psi) 
 return(lprior+llike)
}
  1. The following takes as arguments, the number of iterations N and the innovation standard deviation lambda, and returns (dependent) samples from the target posterior \(\pi(\psi|\boldsymbol{y})\).
rwm<-function(N,lambda)
{
 vec <- rep(0,N)
 psi <- 0
 vec[1] <- psi
 for (i in 2:N) 
 {
  innov <- rnorm(1,0,lambda)
  can <- psi + innov
  laprob <- lpost(can)-lpost(psi)
  u <- runif(1)
  if (log(u) < laprob){ 
   psi <- can
  }
   vec[i] <- psi
  }
 return(vec)
}
