Workshop 1: Metropolis-Hastings

1. Random walk Metropolis. Let’s look at simulating from a \(\text{Gamma}(g,h)\) target distribution (say for fixed \(g=2\) and \(h=2\)) using a random walk Metropolis algorithm with Gaussian \(\text{N}(0,a^2)\) innovations. Copy and paste the following R code chunk into an RStudio script file. Make sure you’re happy with each line of code (especially the log acceptance probability) - if anything doesn’t make sense, ask!

gammaMH<-function(N,g=2,h=2,a) 
{
 vec <- rep(0,N)
 theta <- g/h
 vec[1] <- theta #initialise
 for (i in 2:N) 
 {
  #propose
  can <- theta + rnorm(1,0,a)
  #log acceptance prob
  laprob <- dgamma(can,g,h,log=TRUE)-dgamma(theta,g,h,log=TRUE)
  u <- runif(1)
  if (log(u) < laprob){ 
   theta <- can #accept
  }
   vec[i] <- theta #store
  }
 return(vec)
}

#Run for 10k iters with g=2, h=2, a=1:

out <- gammaMH(N=10000,a=1)
par(mfrow=c(2,1))
plot(ts(out),ylab="Value",xlab="Iteration",main="Trace plot")
hist(out,30,xlab="Value",main="Histogram")
par(mfrow=c(1,1))

(a) Try out the R code above for this simple Metropolis example. See how different choices of the tuning parameter \(a\) affect the mixing of the chain. The example code uses \(a=1\) which isn’t too bad. Try \(a=20\) – this gives a chain which is far too "hot". Very few candidate points are accepted, but when they are, they often represent quite considerable moves within the sample space. Also try \(a=0.1\) – this gives a chain which is far too "cold". Most candidates are accepted, but they represent rather conservative moves within the support of the target density.

(b) As we have seen, high acceptance probabilities tend to mean that the chain is too cold, whereas very low acceptance probabilities correspond to chains which are too hot. Modify the R function so that it records and then prints the overall empirical acceptance rate of the chain (as in Section 2.4.2 of the noets), as well as returning the vector of simulated values. "Tune" the Metropolis sampler by finding a value of \(a\) which results in an overall acceptance rate of around 0.4.

(c) With your tuned value of \(a\), run the sampler for 100,000 iterations, produce a histogram using the resulting samples and overlay the \(\text{Gamma}(2,2)\) density.

2. Random walk Metropolis on the log scale. Let’s look again at simulating from a \(\text{Gamma}(g,h)\) target distribution (as above). Using a properly tuned random walk proposal with Gaussian \(\text{N}(0,a^2)\) innovations for this target (or indeed any such target with non-negative support) can still be inefficient, since proposed values that are negative will be rejected. We cook up a better approach as follows.

Denote the density of the \(\text{Gamma}(g,h)\) target by \(f_{\theta}(\theta)\). The random variable \(\psi=\log \theta\) has density \[f_{\psi}(\psi)=f_{\theta}(e^{\psi})e^{\psi}, \qquad -\infty < \psi < \infty\] using the usual transformation of random variables formula. We now implement the random walk Metropolis sampler with \(\text{N}(0,a^2)\) innovations for the target density \(f_{\psi}(\psi)\) via the following code. Inferences using the target \(f_{\theta}(\theta)\) can then be based on the exponentiated values of \(\psi\). Copy and paste the following R code chunk into an RStudio script file. Make sure you’re happy with each line of code (especially the log acceptance probability) - if anything doesn’t make sense, ask!

log_gammaMH<-function(N,g=2,h=2,a) 
{
 vec <- rep(0,N)
 psi <- log(g/h)
 vec[1] <- psi #initialise
 for (i in 2:N) 
 {
  #propose
  can <- psi + rnorm(1,0,a)
  #log acceptance probability
  laprob <- dgamma(exp(can),g,h,log=TRUE)+can-dgamma(exp(psi),g,h,log=TRUE)-psi
  u <- runif(1)
  if (log(u) < laprob){ 
   psi <- can #accept
  }
   vec[i] <- psi #store
  }
 return(vec)
}

#Run for 10k iters with g=2, h=2, a=1:

out2 <- log_gammaMH(N=10000,a=1)
par(mfrow=c(2,1))
plot(ts(out2),ylab="Value",xlab="Iteration",main="Trace plot")
hist(out2,30,xlab="Value",main="Histogram")
par(mfrow=c(1,1))
